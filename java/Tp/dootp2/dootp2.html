DOO - Canevas du TP 2 : Réalisation d'un type encapsulé

Avertissement :

La classe développée dans ce TP sera utilisée dans le TP suivant. Si le TP est inachevé, il faudra le terminer pour la prochaine séance.

Attendus :
- Développer une classe à partir de spécifications;
- Encapsuler correctement une classe simple;
- Savoir la commenter et la tester au fur et à mesure du développement de chaque méthode;
A rendre en fin de TP :
Déposer sur Arche une archive ZIP (seul format accepté), contenant :

- le compte-rendu (PDF + DOC/ODT),
- les codes sources réalisés (Angle.java et TestAngle.java),
- la documentation HTML complète de la classe Angle.
- Rien d'autre (pas les fichiers .class, ni l'énoncé).

NB : un compte-rendu doit aussi illustrer les notions du cours mises en oeuvre au travers d'exemples tirés du code réalisé par contre, il ne doit pas comporter de portions complètes du code sans commentaire, et surtout pas de copies d'écran (pénalités).

Evaluation :
Sur la base :
- de la méthodologie de développement (évaluée en-ligne),
- du compte-rendu,
- des codes remis.
Pénalités :
- rendu hors Arche,
- archive hors-format, incomplète ou fouillis,
- problème de compilation ou d'interprétation du code,
- similarité avec un autre travail.
Recommandations :
- Attention à bien respecter les noms imposés par l'énoncé, sinon les tests menés par l'enseignant ne seront pas concluants.
- Les fichiers rendus doivent pouvoir être compilés sans erreur (commenter les parties de code défectueuses en cas de doute).
Sujet :

Réaliser, tester et documenter (javadoc + en-tête) un type Angle, correctement encapsulé, à valeurs sur ]-π, π].

L'espace des valeurs du type doit être bien contrôlé.

  • Constructeurs :
    • - un constructeur principal à partir d'une valeur réelle (en radians),
    - un constructeur à partir d'une valeur entière (en degrés),
    - une redéfinition du constructeur par défaut retournant un angle droit,
  • Accesseurs :
    • - une méthode de conversion en chaîne de caractères,
    - une méthode radians qui retourne la valeur de l'angle en radians,
    - une méthode degres qui retourne la valeur arrondie à l'unité la plus proche de l'angle en degrés, et telle que le résultat du test suivant soit satisfait :
    (new Angle ((new Angle (21)).degres ())).degres () == (new Angle (21)).degres ()
    Attention : le choix du type de retour devra être justifié dans le compte-rendu.
    - une méthode somme qui retourne la somme avec un autre angle,
    - une méthode oppose qui retourne l'opposé de l'angle sur le cercle trigonométrique (l'opposé de π/4 est -3π/4).
  • Mutateurs :
    • - une méthode ajouter qui ajoute la valeur d'un autre angle,
    - une méthode inverser qui remplace l'angle par son inverse (l'inverse de π/4 est -π/4).
    NB : ces mutateurs ne doivent pas utiliser les constructeurs.
