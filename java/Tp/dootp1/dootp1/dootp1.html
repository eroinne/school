DOO - Canevas du TP 1 : Classe, attributs et méthodes

Consignes générales :
- Pas d'IDE ni d'éditeur avec complétion automatique dans le cadre des TP de DOO ou QD.
- Tester régulièrement la documentation (commande javadoc --private -d doc).
Attendus :
- Savoir écrire un code Java;
- Savoir développer une classe simple;
- Savoir lui associer des attributs et des méthodes;
- Savoir la commenter et la tester au fur et à mesure du développement de chaque méthode;
- Appréhender les notions de constructeurs, de constructeur principal et de surcharge.
A rendre en fin de TP :
- Le compte-rendu (PDF + DOC/ODT),
- les sources (Pixel.java et TestPixel.java), et la documentation complète de la classe Pixel dans une archive ZIP (seul format accepté).
- Rien d'autre (pas les fichiers .class, ni l'énoncé).
- Remarque : un compte-rendu doit aussi illustrer les notions du cours mises en oeuvre au travers d'exemples tirés du code réalisé par contre, il ne doit pas comporter de portions complètes du code sans commentaire, et surtout pas de copies d'écran (pénalités).
Evaluation :

Dans ce TP, la priorité est mise sur l'apprentissage des concepts objets et de la méthode de travail. L'évaluation portera donc sur la forme du rendu (fichier source qui compile et exécute, compte-rendu sérieux, documentation complète, absence de fichiers parasites) plutôt que sur la quantité de code produite. Le résultat sera un bonus/malus adossé à la note du TP suivant.

Sujet
a) Ecrire une classe Pixel avec deux attributs entiers x et y et les méthodes suivantes :
- un constructeur principal avec deux entiers en paramètres,
- une méthode toString indiquant la valeur du pixel.
b) Ecrire une classe TestPixel avec un main dans lequel on crée un pixel et on l'affiche.
c) Dans la suite du TP, on testera systématiquement les méthodes au fur et à mesure de leur développement.
Compléter la classe Pixel avec les méthodes suivantes :
- un constructeur sans paramètre, construisant le pixel (0,0),
- un constructeur avec un pixel en paramètre,
- un constructeur avec deux réels en paramètres (l'intervalle de troncature est [-0.5, 0.5[),
- un constructeur avec un tableau de deux pixels en paramètre construisant le pixel d'écart entre les deux pixels du tableau : (x2 - x1, y2 - y1),
- une méthode distance2 retournant le carré de la distance euclidienne du pixel à l'origine : (x*x + y*y). Attention au type retourné !
- une méthode plusLoinQue testant si le pixel est plus loin de l'origine que le pixel en paramètre (utiliser distance2).
- une méthode distance2 retournant le carré de la distance euclidienne à un autre pixel (utiliser le constructeur avec un tableau en paramètre).
- une méthode distanceManhattan retournant la distance de Manhattan à un autre pixel,
- une méthode distanceEchiquier retournant la distance de l'échiquer à un autre pixel.
d) Ecrire une classe TestGraphique avec un main dans lequel on crée un pixel et on le trace sur un damier.
Pour le tracé du pixel, on se servira de la classe Damier. Une documentation est fournie dans le répertoire doc. Le programme de test d'utilisation de cette classe, contenu dans le fichier TestDamier.java, s'appuie sur la classe Saisie, également documentée dans le répertoire doc.
e) Dans la suite du TP, on ajoutera aussi des tests graphiques systématiques au fur et à mesure du développement.
Compléter la classe Pixel avec les méthodes suivantes :
- une méthode midpixel retournant le pixel à mi-distance euclidienne d'un autre pixel,
- une méthode midpixelManhattan retournant les pixels à mi-distance de Manhattan d'un autre pixel,
- une méthode midpixelEchiquier retournant les pixels à mi-distance de l'échiquier d'un autre pixel.
