QD - Canevas du TP 1 : Vie et mort des instances

Consignes générales :
- Interdiction d'utiliser un IDE ou un éditeur avec complétion automatique dans le cadre des TP.
- Utiliser la commande javac dans un terminal pour compiler les codes sources java.
- Utiliser la commande java pour interpréter le code JVM (classes).
Attendus :
- Comprendre un code Java;
- Savoir compiler un fichier source;
- Savoir exécuter une classe;
- Lire les retours du compilateur et de l'interpréteur;
- Deviner la nature de Infinity et NaN;
- Découvrir le rôle et les inconvénients d'un ramasse-miettes.
Rendu final :
- Uniquement le compte-rendu : une version PDF et une version source (DOC ou ODT) dans une archive ZIP ou TGZ (seuls formats acceptés).
- Rien d'autre.
- Remarque : un compte-rendu doit impérativement comporter le nom et prénom de l'auteur, la formation suivie, le numéro de groupe TP, le nom de la matière, le thème du TP. Il doit aussi rappeler les attendus.
Evaluation :
Sur la base du compte-rendu. Pénalités pour :
- les rendus hors Arche; vérifier le bon fonctionnement du dépôt dès que possible.
- les rendus incomplets ou en excès, ou dans un format d'accès compliqué pour l'enseignant.
Exercice 1 : Compilation et exécution (environ 1 heure)
Le code du fichier ProgrammeStupide.java comporte onze erreurs.
- Compiler ce programme : javac "nom_du_fichier". Consigner la commande correspondante dans l'énoncé.
- Corriger les erreurs à la compilation une par une dans l'ordre dans lequel elles sont signalées. Les reporter dans le compte-rendu en indiquant précisément la nature de l'erreur (problème de syntaxe ? initialisation ? conflit statique / non statique ? ...) et la solution mise en place. Pour régler les conflits de contextes statique / non statique, privilégier la solution la plus orientée objet.
- Exécuter ce programme : java "nom_de_la_classe". Consigner la commande correspondante dans l'énoncé.
- Corriger les erreurs à l'exécution. S'assurer en particulier que les tests fonctionnent.
- Expliquer dans le compte-rendu à quoi correspondent les mots infinity et NaN.
Exercice 2 : Terminaison des objets (environ 20 minutes)
Avertissement : le comportement du programme peut dépendre du contexte d'exécution. En cas de soucis, utiliser un ordinateur de la salle de TP.
Etudier le fichier Terminator.java.
- La méthode finalize est appelée au moment où l'objet est détruit par le ramasse-miettes de Java.
- Quand on entre la valeur 1, on crée 10 terminators et 10 gros objets avec; comme ils ne sont pas stockés dans des variables, ils sont immédiatement à la merci du ramasse-miettes.
- Quand on entre la valeur 2, on force la mise en route du ramasse-miettes.
- Quand on entre la valeur 0, on arrête le programme.
Compiler et exécuter ce programme.
- Ne pas tenir compte du possible message de déprécation.
- Observer ce qui se passe quand on entre la valeur 1 plusieurs fois de suite.
- Si rien ne se passe, augmenter la taille des gros objets en relançant le programme avec en argument un facteur multiplicateur, par exemple : java Terminator 100
- Tester aussi l'effet de la touche 2.
Consigner dans le compte-rendu :
- le fonctionnement du code et son objectif (que met-il en évidence ?),
- comment le non contrôle du déclenchement du ramasse-miettes est mis en évidence,
- comment le non contrôle de l'ordre dans lequel les objets sont détruits est mis en évidence.
